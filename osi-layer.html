Model Referensi OSI (7 Layer)

Model OSI (Open Systems Interconnection) adalah kerangka kerja konseptual yang digunakan untuk memahami komunikasi jaringan melalui tujuh lapisan yang berbeda.

Layer 7: Application
Layer 6: Presentation
Layer 5: Session
Layer 4: Transport
Layer 3: Network
Layer 2: Data Link
Layer 1: Physical
7

Application Layer

Lapisan Aplikasi - Interface Pengguna

üéØ Fungsi Utama

  • Menyediakan layanan jaringan langsung ke aplikasi pengguna
  • Menangani autentikasi dan enkripsi
  • Mengelola sumber daya jaringan
  • Menyediakan antarmuka pengguna
  • File sharing dan printing

üìã Protokol & Layanan

  • HTTP/HTTPS - Web browsing
  • FTP - File transfer
  • SMTP/POP3 - Email
  • DNS - Domain resolution
  • Telnet - Remote login
  • SSH - Secure shell
  • SNMP - Network monitoring

üíª Contoh Aplikasi

  • Web Browser (Chrome, Firefox)
  • Email Client (Outlook, Gmail)
  • FTP Client (FileZilla)
  • Instant Messaging (WhatsApp)
  • File Sharing (Google Drive)

Poin Penting Layer 7

  • Satu-satunya layer yang berinteraksi langsung dengan pengguna
  • Bertanggung jawab untuk user authentication
  • Menyediakan aplikasi dan services kepada user
6

Presentation Layer

Lapisan Presentasi - Translasi Data

üéØ Fungsi Utama

  • Enkripsi dan dekripsi data
  • Kompresi data
  • Konversi format data
  • Translasi bahasa jaringan
  • Penyeragaman format data

üìã Protokol & Format

  • SSL/TLS - Enkripsi
  • JPEG, GIF, PNG - Gambar
  • ASCII, EBCDIC - Text encoding
  • MPEG, AVI - Video codec
  • MIDI, MP3 - Audio codec

üíª Contoh Layanan

  • Encryption (OpenSSL)
  • Data compression (ZIP)
  • Character encoding (UTF-8)
  • Video streaming codec
  • Image format conversion

Poin Penting Layer 6

  • Sering disebut "Translator" dari OSI model
  • Memastikan data dalam format yang dapat dibaca receiver
  • Menangani encryption, compression, dan translation
5

Session Layer

Lapisan Sesi - Manajemen Koneksi

üéØ Fungsi Utama

  • Membuat, memelihara, dan mengakhiri dialog
  • Manajemen sesi komunikasi
  • Sinkronisasi data exchange
  • Dialog control (half-duplex/full-duplex)
  • Pemulihan sesi

üìã Protokol & Layanan

  • NetBIOS - Network BIOS
  • NFS - Network File System
  • SAP - Session Announcement
  • X.225 - ISO Session
  • SOCKS - Secure socket

üíª Contoh Aplikasi

  • Video Conference (Zoom)
  • Remote session (RDP)
  • Database sessions
  • Terminal emulation
  • VoIP sessions

Poin Penting Layer 5

  • Bertanggung jawab untuk session setup, maintenance, dan termination
  • Memberikan dialog control mekanisme
  • Memungkinkan seamless communication antara devices
4

Transport Layer

Lapisan Transportasi - Pengiriman Data

üéØ Fungsi Utama

  • End-to-end delivery pengiriman data
  • Kontrol aliran data (Flow Control)
  • Error handling dan recovery
  • Multiplexing/Demultiplexing
  • Port management

üìã Protokol Utama

  • TCP - Reliable, connection-oriented
  • UDP - Unreliable, connectionless
  • SCTP - Stream control
  • DCCP - Datagram congestion
  • QUIC - UDP-based modern protocol

üíª Port Contoh

  • Port 80 - HTTP
  • Port 443 - HTTPS
  • Port 22 - SSH
  • Port 25 - SMTP
  • Port 53 - DNS

Poin Penting Layer 4

  • Pertama layer yang menyediakan end-to-end delivery
  • TCP dan UDP adalah protokol utama di layer ini
  • Menangani flow control dan error detection
3

Network Layer

Lapisan Jaringan - Routing

üéØ Fungsi Utama

  • Routing packet ke destinasi
  • IP addressing dan subnetting
  • Logical addressing
  • Path determination
  • Packet forwarding

üìã Protokol Utama

  • IP (IPv4, IPv6)
  • ICMP - Internet Control Message
  • IGMP - Internet Group Management
  • IPsec - IP Security
  • Routing (RIP, OSPF, BGP)

üíª Perangkat Network

  • Router - Routing traffic
  • Firewall - Filtering packet
  • Layer 3 Switch
  • IP gateway
  • Proxy server

Poin Penting Layer 3

  • Bertanggung jawab untuk routing dan forwarding
  • Menggunakan IP address untuk logical addressing
  • Router adalah perangkat utama di layer ini
2

Data Link Layer

Lapisan Penghubung Data - MAC Address

üéØ Fungsi Utama

  • MAC addressing (Physical addressing)
  • Frame creation dan framing
  • Error detection
  • Physical link management
  • Switch operation

üìã Protokol Utama

  • Ethernet - LAN standard
  • PPP - Point-to-Point
  • Frame Relay
  • WiFi (802.11)
  • ARP - Address Resolution

üíª Perangkat Network

  • Switch - Frame forwarding
  • Bridge - Segment network
  • Network Interface Card
  • Access Point (WiFi)
  • Repeater/Hub (L1)

Poin Penting Layer 2

  • Menggunakan MAC address untuk physical addressing
  • Switch adalah perangkat utama di layer ini
  • Menangani frame organization dan error detection
1

Physical Layer

Lapisan Fisik - Hardware & Media

üéØ Fungsi Utama

  • Transmisi raw bit stream
  • Definisi media transmisi
  • Electrical/optical signals
  • Physical topology
  • Connectors dan cables

üìã Media & Standard

  • Copper (Twisted pair, Coaxial)
  • Fiber Optic
  • Wireless (RF, Microwave)
  • USB, Serial, Parallel
  • X.25, V.24 standards

üíª Perangkat & Komponen

  • Network Interface Card (NIC)
  • Hub - Signal regeneration
  • Repeater - Signal amplification
  • Cables - RJ45, Fiber, Coax
  • Modems - Signal conversion

Poin Penting Layer 1

  • Paling bawah layer dalam OSI model
  • Menangani electrical dan mechanical aspects
  • Hub dan Repeater adalah perangkat utama

üí° Contoh Praktis Komunikasi OSI Layer

Contoh 1: Membuka Website Google.com

Skenario: User membuka browser dan akses google.com

Proses Layer-by-Layer:

  • Layer 7 (Application): Browser (Chrome/Firefox) menginterpretasi URL google.com dan membuat HTTP request
  • Layer 6 (Presentation): Data dienkripsi dengan HTTPS/SSL dan diformat dalam bentuk yang sesuai
  • Layer 5 (Session): Session diinisialisasi antara client dan server Google
  • Layer 4 (Transport): TCP membuat koneksi reliable ke server (port 443 untuk HTTPS)
  • Layer 3 (Network): IP routing menentukan path ke server Google menggunakan IP address
  • Layer 2 (Data Link): Ethernet frames dibuat dengan MAC address gateway/router
  • Layer 1 (Physical): Data dikirim dalam bentuk electrical signals melalui kabel atau WiFi

Contoh 2: Mengirim Email

Skenario: User mengirim email melalui Gmail

Proses:

  • Layer 7: Gmail interface ‚Üí User compose email
  • Layer 6: Email content dikompres dan dienkripsi
  • Layer 5: SMTP session dibuat ke mail server
  • Layer 4: TCP port 587 (SMTP) digunakan untuk reliable delivery
  • Layer 3: IP packet diformat untuk routing ke mail server
  • Layer 2: MAC address ditambahkan di frame header
  • Layer 1: Email dikirim via kabel/WiFi dalam bentuk bits

Contoh 3: Video Streaming (YouTube)

Skenario: Menonton video di YouTube

Proses:

  • Layer 7: YouTube app/website menampilkan video player
  • Layer 6: Video stream diformat (H.264, H.265, VP9) dan dikompres
  • Layer 5: RTP (Real-time Transport Protocol) session dibuat
  • Layer 4: UDP digunakan untuk faster delivery daripada TCP
  • Layer 3: IP routing mengirim packets dari YouTube server
  • Layer 2: Ethernet frames dengan MAC address
  • Layer 1: Video bits ditransmisikan melalui network media

Contoh 4: Online Gaming

Skenario: Bermain game online multiplayer

Proses:

  • Layer 7: Game client & server berinteraksi untuk game logic
  • Layer 6: Game data dikompres dan dienkripsi untuk security
  • Layer 5: Game session dipertahankan selama gameplay
  • Layer 4: TCP untuk important data, UDP untuk real-time gameplay data
  • Layer 3: IP routing untuk low latency path
  • Layer 2: MAC addressing untuk local network segment
  • Layer 1: Kecepatan tinggi transmission untuk minimal latency

üì¶ Enkapsulasi Data di Setiap Layer

Ketika data bergerak down dari Layer 7 ke Layer 1, setiap layer menambahkan header-nya sendiri. Proses ini disebut enkapsulasi:

Proses Enkapsulasi (Down the Stack)

Layer 7 (Application): [HTTP Data] ‚Üì Layer 6 (Presentation): [Presentation Header | HTTP Data] ‚Üì Layer 5 (Session): [Session Header | Presentation Header | HTTP Data] ‚Üì Layer 4 (Transport): [TCP Header | Session Header | HTTP Data] = SEGMENT ‚Üì Layer 3 (Network): [IP Header | TCP Header | Session Header | HTTP Data] = PACKET ‚Üì Layer 2 (Data Link): [MAC Header | IP Header | TCP Header | HTTP Data | MAC Trailer] = FRAME ‚Üì Layer 1 (Physical): Bits dikirim melalui kabel/wireless medium

Proses De-enkapsulasi (Up the Stack)

Ketika data diterima, setiap layer melepas headernya (de-encapsulation):

Layer 1 (Physical): Bits diterima dan dikonversi ‚Üë Layer 2 (Data Link): MAC Header diperiksa dan dihapus ‚Üí FRAME ‚Üë Layer 3 (Network): IP Header diperiksa dan dihapus ‚Üí PACKET ‚Üë Layer 4 (Transport): TCP Header diperiksa dan dihapus ‚Üí SEGMENT ‚Üë Layer 5 (Session): Session Header dihapus ‚Üë Layer 6 (Presentation): Presentation Header dihapus dan data di-decode ‚Üë Layer 7 (Application): Original HTTP Data siap digunakan oleh aplikasi

‚öñÔ∏è Perbandingan: OSI Model vs TCP/IP Model

üìä OSI Model

  • 7 Layer
  • Lebih teoritis
  • Dikembangkan oleh ISO
  • Layer 1-5 untuk network
  • Layer 6-7 untuk application
  • Lebih detail dan comprehensive

üåê TCP/IP Model

  • 4-5 Layer
  • Lebih praktis
  • Dikembangkan oleh DoD
  • Layer 1-3 untuk network
  • Layer 4 untuk application
  • Lebih sederhana dan efficient

Mapping OSI ke TCP/IP:

TCP/IP Layer OSI Layers Fungsi Utama
Application 7, 6, 5 User services & communication protocols
Transport 4 End-to-end communication & reliability
Internet 3 Routing & logical addressing
Link 2, 1 Physical transmission & MAC addressing

üîí Security Considerations di Setiap Layer

Layer 1 (Physical) Security

  • Threats: Cable tapping, signal interception, physical damage
  • Mitigation: Fiber optic cables, secure data centers, access control

Layer 2 (Data Link) Security

  • Threats: MAC spoofing, ARP spoofing, switch port security
  • Mitigation: Port security, DHCP snooping, ARP inspection

Layer 3 (Network) Security

  • Threats: IP spoofing, DoS attacks, route hijacking
  • Mitigation: Firewall, IPsec, ingress/egress filtering

Layer 4 (Transport) Security

  • Threats: TCP/UDP port scanning, session hijacking
  • Mitigation: Stateful firewall, intrusion detection, DPI

Layer 5-7 (Upper Layers) Security

  • Threats: SQL injection, XSS, authentication bypass, malware
  • Mitigation: WAF, input validation, encryption, secure coding

üéì Konsep Penting OSI Model

Fundamental Principles:

  • Layering: Setiap layer menyediakan services ke layer di atasnya
  • Abstraction: Layer bawah menyembunyikan kompleksitas dari layer atas
  • Encapsulation: Data dibungkus dengan header di setiap layer
  • Separation of Concerns: Setiap layer punya tanggung jawab spesifik
  • Standardization: Memungkinkan interoperability antar devices

Mnemonics untuk Mengingat OSI Layers:

"Please Do Not Throw Sausage Pizza Away"

  • Physical (Layer 1)
  • Data Link (Layer 2)
  • Network (Layer 3)
  • Transport (Layer 4)
  • Session (Layer 5)
  • Presentation (Layer 6)
  • Application (Layer 7)

Data Units di Setiap Layer:

  • Layer 1: Bit
  • Layer 2: Frame
  • Layer 3: Packet
  • Layer 4: Segment (TCP) / Datagram (UDP)
  • Layer 5-7: Data/Message
