üìã Penetration Testing Phases

Penetration testing adalah proses sistematis untuk mengidentifikasi dan exploit vulnerabilities dalam sistem dengan izin. Terdiri dari 5-7 phases utama tergantung metodologi yang digunakan.

Phase 1: Reconnaissance (Recon)

üîé Passive Information Gathering

Mengumpulkan informasi tanpa direct contact dengan target:

  • WHOIS lookup untuk domain information
  • Google dorking untuk exposedfiles/pages
  • Shodan untuk publicly accessible devices
  • LinkedIn untuk employee enumeration
  • Archive.org (Wayback Machine) untuk historical data
  • DNS enumeration dengan dig/nslookup
  • Social engineering recon (phone calls, emails)

    • üîç Active Enumeration

    Directly probing target untuk gather intelligence:

  • Port scanning (Nmap)
  • Service version detection
  • Subdomain enumeration
  • Web application fingerprinting
  • Network topology mapping
  • Firewall/WAF detection

    • Recon Tools:

    • Nmap - Port scanning
    • TheHarvester - Email/subdomain gathering
    • Shodan - Internet-wide scanning
    • whois/dig - DNS information
    • Metasploit - Information gathering modules

    Phase 2: Scanning & Enumeration

    Network Scanning

    Mengidentifikasi hosts, ports, dan services yang aktif:

  • Port Scanning: TCP/UDP ports (Nmap -p-, -sV)
  • OS Fingerprinting: Determine OS type dan version
  • Service Enumeration: Aplikasi apa yang berjalan, versi berapa
  • Vulnerability Scanning: Gunakan Nessus/OpenVAS untuk automated scanning

    • Web Application Scanning

    Deep dive into web technologies:

  • Web server enumeration (Apache, IIS, Nginx)
  • Web application firewall (WAF) detection
  • Directory/file enumeration (Gobuster, dirbuster)
  • Parameter discovery
  • Technology stack identification
  • CMS/Framework detection

    • Phase 3: Vulnerability Assessment

    Vulnerability Identification

    Mengidentifikasi semua potential vulnerabilities dari scan results:

  • Analyze scan results dari automated tools
  • Manual testing untuk logic flaws
  • Business logic vulnerabilities
  • Configuration weaknesses
  • Unpatched systems dan outdated software
  • Default credentials
  • Weak encryption

    • Vulnerability Scoring:

    • CVSS (Common Vulnerability Scoring System)
    • CVSS v3.1 adalah standard saat ini
    • Score 0-10 (0=None, 10=Critical)
    • Pertimbangkan impact terhadap bisnis

    Phase 4: Exploitation

    Exploiting Vulnerabilities

    Membuktikan vulnerabilities dengan actual exploitation:

  • Proof of Concept (PoC) untuk setiap vulnerability
  • Gunakan Metasploit, custom scripts, atau manual exploitation
  • Dokumentasi setiap successful exploit
  • Hindari causing damage atau system downtime
  • Stay within scope of engagement
  • Focus pada high-risk vulnerabilities first

    • Post-Exploitation

    Setelah initial access, perform deeper exploration:

  • Privilege escalation (obtain higher privileges)
  • Persistence (maintain access untuk future testing)
  • Lateral movement (access other systems)
  • Data exfiltration (retrieve sensitive data)
  • Credential harvesting
  • Covering tracks (cleanup logs)

    • Phase 5: Reporting & Remediation

    Comprehensive Report

    Detail findings dalam professional report:

  • Executive Summary: High-level overview untuk management
  • Methodology: Tools & techniques yang digunakan
  • Vulnerability Details: Setiap finding dengan PoC
  • Business Impact: Risk assessment terhadap bisnis
  • Remediation Recommendations: How to fix setiap issue
  • Timeline: Kapan vulnerabilities harus diperbaiki
  • Appendix: Technical details, screenshots, scan results

    • Report Structure:

    • Clear, professional language (tidak technical jargon untuk executives)
    • Screenshots & PoC code
    • CVSS scores & risk ratings
    • Detailed remediation steps
    • Prioritized by risk

    üìú Legal & Compliance

  • Rules of Engagement (RoE): Written agreement sebelum testing
  • Scope Definition: Jelas target systems & testing windows
  • Authorization: Signed permission dari authorized personnel
  • NDA: Non-disclosure agreement untuk hasil findings
  • Compliance: Follow applicable laws & regulations
  • Data Protection: Secure handling of sensitive data

    • üéØ Technical Best Practices

  • Backup Systems: Backup sebelum testing critical systems
  • Time Windows: Testing hanya dalam agreed-upon windows
  • No Disruption: Avoid causing system outages/data loss
  • Clean Up: Remove test artifacts & reverse changes
  • Documentation: Document semua activities & findings
  • Escalation Procedures: Know when to stop & report issues
